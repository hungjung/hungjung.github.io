Web架構概觀

  • 原文出處： Web Architecture 101

  1. DNS協定
    • At the most basic level DNS provides a key/value lookup from a domain name (e.g., google.com) to an IP address (e.g., 85.129.83.120), which is required in order for your computer to route a request to the appropriate server.
  2. 負載平衡器(Load Balancer)
    • horizontal application scaling： 除了既有的機器外，再增加其他機器一起運作
    • vertical application scaling： 在既有的機器上加記憶體、加CPU
  3. Web應用伺服器(web app servers)
    • They execute the core business logic that handles a user’s request and sends back HTML to the user’s browser.
  4. 資料庫伺服器(Database server)
    • SQL
    • NoSQL
  5. 快取服務(Caching Service)
    • A caching service provides a simple key/value data store that makes it possible to save and lookup information in close to O(1) time.
    • The two most widespread caching server technologies are Redis and Memcache.
  6. 任務排程(Job Queue & Servers)
    • Most web applications need to do some work asynchronously behind the scenes that’s not directly associated with responding to a user’s request.
    • two components:
      • a queue of “jobs” that need to be run
      • one or more job servers (often called “workers”) that run the jobs in the queue.
  7. 全文檢索(Full-text Search Service)
    • The most popular full-text search platform today is Elasticsearch though there are other options such as Sphinx or Apache Solr.
    • Oracle中，可以使用 Instr 函數對某個字串進行判斷，判斷其是否含有指定的字元。其語法為：Instr(string, substring, position, occurrence) <出處>
  8. 在web app server上所提供的眾多服務(Services)
    • 使用者帳戶及存取權限管理 (Account service)
    • 網頁內容維護管理 (Content service)
    • 付款服務 (Payment service)
    • HTML轉換成PDF輸出的服務 (HTML → PDF service)
  9. 資料(Data)
    • a data pipeline to ensure that data can be collected, stored, and analyzed.
    • A typical pipeline has three main stages:
      • The app sends data 主要是使用者日常運作的交易資料
      • The raw data as well as the final transformed/augmented data
      • The transformed/augmented data is often loaded into a data warehouse for analysis.
  10. 雲端儲存設備(Cloud storage)
  11. CDN
    • CDN stands for “Content Delivery Network” and the technology provides a way of serving assets such as static HTML, CSS, Javascript, and images over the web much faster than serving them from a single origin server.
